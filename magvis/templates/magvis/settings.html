🧠 KI-Modelle

Welche LLM/Bild-Modelle Magvis für die Inhaltsgenerierung nutzt.

{{ form.glm_model }}
Verwendung: Erzeugt Blog-Texte, Quiz-Fragen, Mini-Spiel, FAQs, Gravur-Vorschläge, Auto-Titel/-Beschreibungen für Social-Media-Posts.
Beispiele: glm-5.1 (Standard, beste Qualität), glm-4.6, glm-4-plus, glm-4-air (günstiger).
API-Key: wird aus user.zhipu_api_key gelesen (in API-Einstellungen setzen).
{{ form.glm_base_url }}
OpenAI-kompatibler Endpoint. Standard: https://api.z.ai/api/coding/paas/v4 (z.AI Coding-Plan).
Nur ändern, wenn du einen anderen Provider/Endpoint nutzt (z.B. eigener BigModel-Account, OpenRouter, etc.).
{{ form.gemini_image_model }}
Verwendung: Erzeugt Diagramm-Bilder & Brainstorming-Bilder im Blog, optional auch KI-Text-Overlays beim Bild-Posting.
Empfehlungen (von günstig nach hochwertig):
  • gemini-2.5-flash-image — günstig, schnell, gute Qualität (Standard, ~$0.039/Bild)
  • gemini-3-pro-image-preview — höchste Qualität, langsamer, teurer
  • imagen-4.0-fast-generate-001 — schnelle Imagen 4 Variante
  • imagen-4.0-generate-001 — Standard Imagen 4
  • imagen-4.0-ultra-generate-001 — beste Imagen-Qualität
Für Magvis-Blog-Bilder reicht gemini-2.5-flash-image völlig.

🎬 Video-Defaults (vidgen)

Standardwerte für die Video-Generierung in Stage 1. Stack: Modal Remotion + Pexels-Stock-Clips + OpenAI TTS + Lyria-Musik (kein RunPod).

{{ form.default_video_template }}
Bestimmt Skript-Stil und Aufbau. Beispiele:
  • youtube_short — vertikales 9:16 Format für Shorts (empfohlen für Magvis)
  • job_intro — Beruf-Vorstellung mit Hook + Aufgaben + Gehalt
  • product — Produkt-Spotlight
  • tutorial — How-To
  • facts — Wissens-Snippets
  • hook_story — Storytelling mit starkem Hook
{{ form.default_video_duration }}
Empfohlen: 30-60 Sekunden für YouTube Shorts / TikTok / Reels.
Längere Videos (90s+) kosten mehr Modal-Render-Zeit.
{{ form.default_voice }}

  • alloy — neutral, ausgewogen (empfohlen für Naturmacher)
  • echo — männlich, klar
  • fable — britischer Akzent
  • onyx — tiefe, ruhige Männerstimme
  • nova — junge Frauenstimme
  • shimmer — warme Frauenstimme

📝 Blog-Defaults

Standardwerte für die Blog-Erstellung in Stage 4. Naturmacher-Stil mit Quiz, Mini-Spiel und FAQs.

{{ form.default_quiz_questions }}
Anzahl Multiple-Choice-Fragen im Mini-Quiz. Empfohlen: 3-5 (mehr ist anstrengend für Leser).
{{ form.default_faq_count }}
FAQs am Ende des Beitrags (mit JSON-LD Schema für Google Rich Snippets). Empfohlen: 4-6.
{{ form.naturmacher_brand_voice }}
System-Prompt-Zusatz, den GLM bei jeder Text-Generierung erhält.
Beispiel (Naturmacher.de):

Tipps: Nenne Marke, Tonalität (du/Sie/neutral), Stil-Keywords (warm/sachlich/witzig), no-go-Phrasen.

📦 Produkt-Defaults (ploom)

Stage 3 erzeugt 2 Produkte parallel mit je 4 KI-Bildern (Studio, Lifestyle, Geschenk-Übergabe, Macro) + 3 fixe CDN-Bilder.

{{ form.default_ploom_settings }}
Verweist auf eine vorhandene PLoomSettings-Konfiguration (unter P-Loom-Einstellungen).
Liefert: Referenz-Topf-Foto (Pflicht für KI-Bildgenerierung), Topf-Beschreibung (z.B. "14cm × 14cm Keramik"), Shopify-Store (für Veröffentlichung), Default-Preise und Tags.
Wichtig: Ohne diese Verknüpfung kann Stage 3 keine Produkte erstellen.
{{ form.fixed_cdn_image_urls_text }}
Eine URL pro Zeile. Diese Bilder werden bei jedem Produkt als Phase-2-Bilder eingefügt (zwischen den 3 ersten KI-Bildern und dem letzten Detail-KI-Bild).
Zweck: immergleiche Stimmungsbilder (z.B. Marken-Erklärbilder, Lifestyle-Hero-Aufnahmen), die jedes Produkt mit dem Markenkontext unterlegen.
Beispiele für Naturmacher: 
https://naturmacher.de/cdn/shop/files/Dein_Pflanzmoment_1.png
https://naturmacher.de/cdn/shop/files/Blumentopf_mit_Gravur_xyz.png
https://naturmacher.de/cdn/shop/files/Warum_ein_Naturmacher-Topf_so_besonders_ist_2.png
Lade in Shopify hoch (Inhalt → Dateien) und kopiere die CDN-URL.

🤖 Auto-Run (Themen-Queue)

Lasse Magvis automatisch in regelmäßigen Abständen das nächste Topic aus deiner Themen-Queue ziehen und einen kompletten Workflow durchlaufen.

{{ form.auto_run_enabled }}
Wenn aktiv, läuft die auto_run_from_queue Cron-Task gemäß Schedule und startet einen Workflow, sofern in deiner Themen-Queue ein pending-Topic vorhanden ist.
{{ form.auto_run_schedule }}
Standard Cron-Expression. Beispiele:
  • 0 6 * * * — täglich um 06:00 Uhr
  • 0 9 * * 1,4 — montags und donnerstags um 09:00
  • 0 */6 * * * — alle 6 Stunden
  • 30 18 * * 0 — sonntags um 18:30
💡 Hinweis: Das Cron-System muss in Celery-Beat eingerichtet sein (System-Admin-Aufgabe).
Abbrechen → Report-Konfiguration

📚 Vor dem ersten Run prüfen

  1. ✅ API-Keys gesetzt: zhipu_api_key (GLM), gemini_api_key, openai_api_key, Upload-Post-Key
  2. ✅ P-Loom Settings: Referenz-Topf-Foto hochgeladen, Shopify-Store verknüpft, Topf-Beschreibung gesetzt
  3. ✅ Magvis-Settings (diese Seite): P-Loom Settings ausgewählt, 3 fixe CDN-URLs eingetragen
  4. ✅ Report-Konfiguration: Empfänger-E-Mail eingetragen
  5. ✅ Themen-Queue: mindestens 1 Topic angelegt
