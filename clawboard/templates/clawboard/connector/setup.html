2
Dateien platzieren

Lege beide Dateien in einen Ordner deiner Wahl. Die Config-Datei muss im selben Ordner liegen oder unter ~/.clawdbot/.

Empfohlene Ordnerstruktur 
~/clawboard/
├── connector.py
└── clawboard-connector.json
Oder mit Standard-Pfad 
# Config in Standard-Pfad legen (wird automatisch gefunden)
mkdir -p ~/.clawdbot
mv clawboard-connector.json ~/.clawdbot/
3
Optional: System-Metriken aktivieren

Installiere psutil, damit der Connector CPU, RAM und Speicherplatz-Daten senden kann. 

pip install psutil
4
Connector starten

Starte den Connector - er sendet ab sofort alle 30 Sekunden Daten an dein Clawboard Dashboard.

Wenn Config im selben Ordner liegt 
python3 connector.py --config clawboard-connector.json
Wenn Config unter ~/.clawdbot/ liegt 
python3 connector.py
Im Hintergrund laufen lassen 
nohup python3 connector.py --config clawboard-connector.json &
So erkennst du, dass es funktioniert

Im Terminal siehst du Push #1 erfolgreich und im Dashboard erscheinen System-Daten, sobald der Connector läuft.

Config-Vorschau

So sieht deine Config aus (Token ist vorausgefüllt): 

{
  "push_url": "https://www.workloom.de/clawboard/api/push/",
  "connection_token": "{{ active_connection.gateway_token }}",
  "heartbeat_interval": 30,
  "reconnect_delay": 10,
  "workspace": "~/clawd"
}
workspace = Ordner in dem deine Memory-Dateien (.md) liegen. Der Connector liest .md Dateien aus workspace/memory/ und sendet sie ans Dashboard.
